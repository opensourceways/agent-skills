# Shared Skills

这个目录包含所有团队共享的通用 skills。

## 原则

放在此目录的 skills 应该：
- 对多个团队都有价值
- 功能通用，不包含团队特定逻辑
- 经过充分测试和文档化
- 由至少两个团队成员审查通过

## Skills 列表

当前可用的共享 skills：

- [example-skill.md](./example-skill.md) - Git 提交助手示例

## 贡献指南

如果你想将团队 skill 提升为共享 skill：

1. 确保 skill 已经在团队内使用并验证
2. 移除任何团队特定的配置或逻辑
3. 添加完整的文档和使用示例
4. 创建 PR 并邀请其他团队成员审查
5. 获得至少2个团队成员的批准后合并

## 维护

共享 skills 由所有团队共同维护。如果发现问题或有改进建议，请：
- 提交 Issue 说明问题
- 或直接提交 PR 修复

## 弃用流程

如果某个共享 skill 不再需要：
1. 在文件开头添加 `[DEPRECATED]` 标记
2. 说明弃用原因和替代方案
3. 保留至少一个月（或一个发布周期）
4. 最终移除文件并更新本 README
